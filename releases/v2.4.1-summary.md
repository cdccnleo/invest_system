```
推送源 3 路: bridge_to_web_ui() / L3Advisor / intraday_hermes_agent
       ↓
  PG l3.push_notification_log (持久化)
       ↓ pg_notify
  PG LISTEN pnl_channel
       ↓
  HermesWebSocketServer (asyncio + websockets 15.0.1)
       ↓ broadcast
  客户端 2 类:
    - Streamlit (st.components.v1.html JS 客户端, 1-2s 收到)
    - Telegram bot / 浏览器
```

## 二十、⭐ 重要 PIT 反思 (B3 类变量污染)

**PIT #34 是 v2.4.1 实施中最严重 (⭐ 严重级) 的坑**, 必须全局警觉:

```python
# ❌ 错: 类变量
class HermesWebSocketServer:
    _stop_event_sync = False  # 所有实例共享!

    def _pg_listen_sync(self, callback):
        while not self._stop_event_sync:  # 看类变量
            ...

    async def stop(self):
        HermesWebSocketServer._stop_event_sync = True  # 污染所有实例
```

**现象**: 测试中跑 2 个 server 实例, server#1 stop 后, server#2 起来的 PG listener 看到类变量是 True, **永不订阅 pnl_channel**, NOTIFY 触发不了广播.

**修复**:
```python
# ✅ 对: 实例变量
def __init__(self, ...):
    import threading
    self._stop_event_sync_instance = threading.Event()

def _pg_listen_sync(self, callback):
    stop_event = self._stop_event_sync_instance  # 实例
    while not stop_event.is_set():
        ...

async def stop(self):
    self._stop_event_sync_instance.set()  # 只影响本实例
```

**PIT 教训** (⭐ 严重, 全局适用):
1. **类变量绝不能用于跨实例状态** (stop 标志 / 计数器 / 配置 / cache)
2. **测试中反复 `start/stop` 同一类多个实例时, 第一个 stop 必污染后续**
3. 用 `threading.Event` / `asyncio.Event` 实例化
4. 同样适用于: 单例模式 `_instance` / 类级 `_counter` / 类级 `_config`
5. **实战必检**: 任何类里**有下划线开头 + 类型暗示状态**的变量, 都应该用 `self.xxx = ...` 在 `__init__` 里实例化

**复盘**: V24-B3 实施 1.5h 中, 30% 时间花在调试 PIT #34 (类变量污染) + PIT #35 (gather 返回值顺序) 上. 真正的架构设计 (PG NOTIFY + WS 广播 + 3 类客户端) 只用 30min.
