## 二十、⭐ 重要 PIT 反思 (B3 类变量污染 + C1 实战 8 PIT)

### 20.1 V24-B3 严重级反思 (PIT #34 ⭐)

详见 v2.4.1 §20 完整复盘

**核心**: **类变量 `_stop_event_sync` 跨实例污染** — 测试中反复 start/stop 同一类多个实例时, 第一个 stop 必污染后续.

**修复**: 用 `threading.Event` / `asyncio.Event` **实例化**

**教训**:
- 类变量只用于 immutable 常量
- 状态变量必须实例化
- pytest 中反复创建实例时要特别小心
- 用 linter 检测类变量用于状态管理的反模式

### 20.2 **v2.4.2 新增: V24-C1 8 PIT 实战反思 (PIT #36-#43)**

详见 `references/v24-c1-integration-pitfalls.md` 完整复盘

**实战关键**:
- **#36 ⭐**: 价格缺失 fallback 0 波动率 (不算 VaR) — 实战持仓 cost_enc 解密失败常见
- **#37 ⭐**: 持仓 0 行返 schema 完整 (不抛异常) — "算 X"函数输入空时必返完整 schema
- **#38 ⭐**: total=0 时返 0/空 schema (不除零) — 组合汇总兜底
- **#40 ⭐**: 触发器去重 (1h 内同 code+type) — 实战触发器 88 条高频, 必须去重
- **#43 ⭐**: 告警频次限制 (1d 10 条 + 1h 1 条 per code) — 实战必须, 否则告警风暴

**总 PIT 教训** (v2.2 21 + v2.3 11 + v2.4 11 + v2.4.2 8 = 43):
- 实战数据缺失时, fallback 永远不抛异常, 返完整 schema
- 任何"算 X"函数, 输入空时必返完整 schema (不抛)
- 触发器必须有去重 + 频次限制
- webhook secret 缺失静默降级, 不阻断主流程
- 实战部署必须有 watchdog + cron 兜底 + 关键路径多级降级
