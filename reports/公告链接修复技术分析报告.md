## 1 问题描述

持仓股公告系统的"查看原文"功能存在链接问题：所有公告的 `🔗 查看原文` 链接均指向同一固定地址 `https://data.eastmoney.com/notices/hot.html`（东方财富公告热点页），而非各公告对应的真实原文详情页。

**影响范围**：所有通过 EastMoney API 采集的公告（占公告总量的绝大多数）。

---

## 2 根因分析

### 2.1 定位

根因位于 [fetch_announcements.py:L150](file:///c:/PythonProject/invest_system/scripts/fetch_announcements.py#L150)：

```python
em_url = f"https://data.eastmoney.com/notices/hot.html"  # art_id 为 None，暂用热点页
```

该行代码注释已明确说明：`art_id 为 None`——即开发时发现 API 返回的 `art_id` 字段为空，因此使用了硬编码回退地址。

### 2.2 深层原因

EastMoney 公告列表 API（`np-anotice-stock.eastmoney.com/api/security/ann`）返回的每条公告数据中：

| 字段 | 说明 | 原代码使用情况 |
|------|------|:---:|
| `art_id` | 被误认为文章ID | ❌ 始终为 `None`（不存在此字段） |
| `art_code` | **真正的文章唯一编码** | ❌ 未使用 |
| `codes` | 嵌套数组，含 `stock_code` | ❌ 未读取 |
| `title` / `notice_title` | 公告标题 | ✅ 已使用 |
| `notice_date` | 公告日期 | ✅ 已使用 |

实际 API 响应结构：

```json
{
  "art_code": "AN202605291823023890",       // ← 真正的文章编码
  "codes": [{
    "stock_code": "300059",                  // ← 股票代码（嵌套在数组中）
    "inner_code": "10581294001978",
    "short_name": "东方财富",
    "market_code": "0"
  }],
  "title": "东方财富: ...",
  "notice_date": "2026-05-29 00:00:00",
  "columns": [{"column_code": "001002002003", "column_name": "限售股份上市流通"}]
}
```

## 3 技术方案

### 3.1 目标 URL 格式

通过测试 3 种候选 URL 模式，确认唯一正确的构造方式：

| 模式 | URL 构造 | HTTP 状态 | 结果 |
|:---:|------|:---:|:---:|
| A | `https://data.eastmoney.com/notices/detail/{stock_code}/{art_code}.html` | 200 | ✅ 直达公告详情页 |
| B | `https://data.eastmoney.com/notices/detail/{inner_code}/{art_code}.html` | 200→302 | ❌ 重定向到热点页 |
| C | `https://np-anotice-stock.eastmoney.com/.../detail?art_code=...` | 200 | ❌ API JSON，非用户可读页 |

**选定模式 A**。

### 3.2 实现代码

```python
# 修复前（fetch_announcements.py L150）
em_url = f"https://data.eastmoney.com/notices/hot.html"

# 修复后
art_code = n.get("art_code", "")
codes_list = n.get("codes", [])
em_stock_code = codes_list[0].get("stock_code", code) if codes_list else code
if art_code:
    em_url = f"https://data.eastmoney.com/notices/detail/{em_stock_code}/{art_code}.html"
else:
    em_url = f"https://data.eastmoney.com/notices/hot.html"  # 回退
```

### 3.3 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| `art_code` 为空时 | 回退热点页 | 兼容极少无 art_code 的公告 |
| `stock_code` 来源 | `codes[0].stock_code` | API 返回的精确股票代码 |
| `stock_code` 回退 | 使用输入参数 `code` | 确保 URL 至少可访问 |
| `ann_id` 格式 | 保持不变（`date_str\|title[:30]`） | 向后兼容，避免 DB 重复 |
